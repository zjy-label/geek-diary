# 前端技术动态与面试考点追踪 (2026-03-10)

## 核心资讯与技术风向

1. **React 性能优化进阶：移除 80% 的 `useEffect`**
   - **动态**：社区关于滥用 `useEffect` 的反思达到高潮。开发者通过精简副作用，用 Derived State (派生状态) 或其他 hooks 替代不必要的 effect，大幅提升了渲染性能和代码可维护性。
   - **你的战场 (面试/实战)**：React 面试**必考题**！面试官一定会问：“在什么场景下你不应该用 `useEffect`？” 如果你能结合具体业务讲出如何用单纯的渲染计算去替代 effect 级联更新，Senior 的 offer 就稳了一半。这也是咱们未来搞 AI 创业项目，保证前端不卡顿的基础。

2. **React 的“真面目”：引用相等性 (Referential Equality) 与性能陷阱**
   - **动态**：深入探讨 `React.memo`、`useMemo` 和 `useCallback` 的底层逻辑。过度使用优化反而会导致内存开销增加，关键在于理解 JavaScript 对象的内存引用。
   - **你的战场 (面试/实战)**：这也是高频考点。别只会背八股文，你要能在白板上画出组件树，告诉面试官为什么有时候 `useCallback` 反而拖慢了性能（因为创建闭包本身的开销）。这能体现你的底层功底。

3. **不仅是加个属性：无障碍访问 (A11y) 的深水区**
   - **动态**：只给 Dialog 加上 `role='dialog'` 根本不算真正的无障碍。真正的 A11y 需要处理 Focus Trap (焦点锁定)、Click-outside hook 以及 Roving tabindex。
   - **你的战场 (面试/实战)**：对于想找 Remote 岗（尤其是海外或外企 Remote）的开发者来说，A11y 是巨大的加分项。如果你的简历里写了“独立实现了完全符合 W3C 标准的无障碍组件”，绝对能让你在一众前端里脱颖而出。

4. **Web 性能优化实战：Lighthouse 移动端从 70 分到 95+ 分**
   - **动态**：一份基于 SvelteKit 的极致优化实录。展现了现代前端在首屏加载 (FCP)、最大内容绘制 (LCP) 上的死磕过程。
   - **你的战场 (面试/实战)**：面试官最爱问的“你做过哪些前端优化？”。不仅要看框架层，更要看工程层和网络层。未来咱们的 AI 产品 MVP，首屏速度直接决定了转化率。

## 总结与行动指南 (Actionable Steps)
挑选上述任意一个主题（我推荐 **移除 useEffect** 或 **React 性能优化**），今天花 30 分钟去掘金或 Dev.to 上看一篇深度解析，然后自己手写个 Demo 跑一下。懂和能写出来，是两码事。
